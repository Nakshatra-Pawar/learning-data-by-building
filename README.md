# Learning Data by Building

A collection of beginner-to-intermediate data engineering and data science projects built step by step using Python, SQL, and practical data workflows across different tools and libraries.

## Purpose

I created this repository to document my learning journey in data engineering and data science through hands-on projects. My goal is to learn in a practical way by solving real problems step by step, instead of only reading concepts.

This repository helps me:

* practice data skills by building small projects
* strengthen my foundation in Python, SQL, and data workflows
* learn how to work with real datasets and different data formats
* improve my problem-solving, debugging, and project-building skills over time

## What this repository includes

This repo will contain a growing collection of learning projects across different areas of data engineering and data science, such as:

* data cleaning and transformation
* ETL and pipeline-style projects
* exploratory data analysis
* working with structured and semi-structured data
* analytics-ready dataset creation
* Python-based and SQL-based problem solving

Each project is added as part of my learning process, so over time this repository will reflect both completed work and my growth as I continue learning.

## Current Project

### Chama Data Transformation

A mini ETL project where nested JSON event data is parsed, flattened, transformed, and exported into analytics-ready CSV files using Python and Pandas.

## Why I made this repo

I am still learning and building my foundation, so I wanted one place where I can keep all my practice projects together in a clean and organized way. This repository is not just for storing solutions, but also for tracking progress, improving project quality, and showing how I learn by building.

## Going forward

I plan to keep adding more projects here as I continue learning. Over time, I want to make the projects more polished by improving code quality, writing better documentation, using more tools and libraries, and building stronger end-to-end data workflows.
